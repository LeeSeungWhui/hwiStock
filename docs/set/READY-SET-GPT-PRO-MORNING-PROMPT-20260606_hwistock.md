# GPT Pro Morning Watchlist Prompt Contract

## 1. Route

This prompt is sent only by:

```text
local Codex CLI -> local browser-use -> user's logged-in local Chrome -> ChatGPT Pro
```

Do not send this prompt through SSH browser-use, hwiServer-side browser
automation, reverse Unix socket Chrome bridges, remote Chrome, or headless
browser automation.

If local Codex CLI or local browser-use is unavailable, the job must write a
named `NO_TRADE` safe-block artifact or a DeepSeek-derived
`morning_watchlist/v0` artifact. It must not silently switch routes.

## 1A. Sunday Rehearsal vs Monday Final

The Sunday rehearsal artifact is useful only to prove that the Monday GPT Pro
morning path can run. It is not a Monday final watchlist. It must stay labeled
as rehearsal/provisional so the operator and Flash document can see that a
Monday refresh is still recommended.

For the late Sunday rehearsal on `2026-06-07` after the missed `07:15 KST`
window, the artifact metadata must be explicit:

```json
{
  "schema_version": "morning_watchlist/v0",
  "target_trade_date_kst": "2026-06-08",
  "generated_at_kst": "2026-06-07T11:33:00+09:00",
  "purpose": "monday_preopen_rehearsal_late",
  "input_window_kst": {
    "start": "2026-06-05T15:00:00+09:00",
    "end": "2026-06-07T11:33:00+09:00"
  },
  "non_trading_day_carryover": true,
  "requires_monday_refresh": true
}
```

The actual Monday final artifact for the `2026-06-08` paper/mock trading day
must be generated by the local Codex CLI/local browser-use/local Chrome route at
`2026-06-08T07:15:00+09:00` or another explicitly accepted final preopen time on
that same trading date when possible.

If Monday final generation is missed but a Sunday rehearsal/candidate artifact
exists for the correct `target_trade_date_kst`, the first Flash bucket must not
silently relabel it as final. It may still proceed through deterministic Flash,
intent, portfolio, order-state, and broker preflight gates, but the trade
document must carry `morning_watchlist_status=provisional`,
`morning_watchlist_refresh_required=true`, and explicit warning reasons.

```text

paper/mock trading experiment. You are advisory only.

Review the attached/supplied sanitized morning input bundle and produce a

Hard constraints:

  universe refs.
```

## 3. Required Local Validation

Before storing GPT Pro output as `morning_watchlist/v0`, the Codex CLI wrapper
or downstream validator must check:

- JSON parse succeeds;
- `route = codex_cli_local_browser_use`;
- `forbidden_actions_acknowledged = true`;
- no credential/account/raw secret text is present;
- every item has source refs or is marked `watch_only` / `reject`;
- no executable order fields are present;
- max item count is respected;
- output is finalized before the first Flash bucket or becomes invalid for that
  bucket.
